# 執筆規約（WRITING_GUIDE）— corona-a5 対応チェックリスト

コロナ社テンプレート（corona-a5-1.1、V1.1）のマニュアルから抽出した、本書執筆時の必須規約（電磁気教科書と共通）。
章を執筆・改稿するエージェントは着手前にこのファイルを必ず読むこと。
一次情報は `publisher/corona-a5-utf8/Manual/manual.pdf`。

## 1. ビルド

- エンジン: **pLaTeX 専用**（LuaLaTeX・upLaTeX 不可）
- パイプライン: `platex → (pbibtex) → (mendex) → platex → dvips → ps2pdf`（`book/build.sh` を使う）
- クラス宣言: `\documentclass[tombow,nosetpagesize]{corona-a5-1.1}`／校正時は `[Proof,MASUME]`
- **corona-a5-1.1.cls の改変禁止**。独自マクロは `book/` 内の別 sty に分離
- graphicx は必ず `\usepackage[dvips]{graphicx}`

## 2. ファイル構成

- 章ファイルは `chapterNN.tex`（番号命名必須。内容名は不可）
- 章ファイルからの `\input` 入れ子は禁止（図・プログラムの読み込みは可）
- まえがき: `\begin{preface}{日付}{著者名}`／付録: `\begin{Appendix}`（`\appendix` は使わない）
- 章末問題解答は巻末 `\begin{Answer}`（章番は `\answer` で進める）
- 文献は巻末 `thebibliography`（章末との併用禁止）

## 3. 本文環境の使い分け

| 用途 | 環境 |
|---|---|
| 定義 | `\begin{定義}[名前]`（罫線なし版は `定義*`） |
| 定理・補題・系 | `定理` `補題` `系`（同上） |
| 例題 | `\begin{例題}[名前]` → 直後に `\begin{解答}`（末尾◇自動） |
| 本文中の例 | `\begin{例}` |
| 本文中の小問 | `\begin{問}`（章でリセット、解答は巻末「問の解答」） |
| 章末問題 | `\begin{章末問題}` + `\item` |
| 注意書き | `\begin{注意}` |
| コラム | `\begin{COLUMN}`（最大見開き2頁。内部のフロートは `[H]`） |
| 箇条書き | `\begin{箇条書}`（コロナ社指定） |

## 4. 数式

- ベクトル・行列は **`\boldsymbol`**（`\mathbf` 禁止 — 講義資料と同じ流儀）
- `cases` 環境**禁止** → `array` + `\right\}`
- 複数式の整列は `eqnarray` 系（`&=&`）。式番なしは `\nonumber`／環境 `*`
- イコール揃えの単独式は `\=` が使える
- 立体ギリシャ小文字は `\alphaup` `\betaup` … `\omegaup`（クラス提供）
- 量記号のギリシャ大文字はイタリック: `\mathit{\Delta}x`
- 行列内の分数は `\tfrac`
- `{\it}` 禁止 → `{\sl}`、本文中の太字イタリックは `{\bi}`
- 付帯条件は式の後ろに全角2倍アキ＋（　）書き

## 5. 句読点・文体

- 句読点は **「，」「。」** に統一（「、」「．」、欧文カンマ禁止）
- 日本語の単語間に余計な半角スペースを入れない
- 「核心」という語は使わない（→「要点」「ポイント」等）
- 章・節参照は `\ref{}` を使い「第」を付けない（例: 「3章で示した」）
- `\cite` は句点の**前**に置く

## 6. 図

- **EPS・原寸（`scale=1`）・`clip` 付き**で読み込む。width/height 指定は避ける
- 図中文字は **11級（footnotesize 相当）**、変数はイタリック。**アウトライン化禁止**
- ファイル名は `fig{章}.{番}.eps`（例: `fig9.3.eps`）。`\label` はファイル名と揃える
- **1図 = 1生成スクリプト**を `book/figures/scripts/` に保存（PNG直接加工・手描き一発は禁止）
  - 概念図・回路図: TikZ standalone（platex+dvips 互換に注意）→ EPS
  - グラフ: matplotlib → EPS（フォント8pt ≒ 11級）
- 必ず `figure` 環境 + `\caption`。**初出は `\FIGref{}`（太字）**、2回目以降 `\ref{}`。表も同様に `\TABref{}`
- 横並びは `Hfloat` 環境 + `tabular`

## 7. 索引・文献

- `\index{よみ@表記}`。数字は0埋め（`\index{02ぶんぎ@2分木}`）、ギリシャ・記号は `$..$@$..$`
- 索引語の対象: 定義した用語・キーワード。単位記号・量記号・社名からは取らない。見出しからは取らず本文から
- mendex + `corona+.ist`（和欧分離が必要になったら `corona.ist` に切替え）
- 文献書式: `\bibitem{key}著者名：書名，pp. NN，出版社（年）`。BibTeX なら `corona_standard.bst`

## 8. 禁止・注意（ハマりどころ）

- `\verb` は見出し・脚注・図説・問題/解答環境内で使用不可
- COLUMN 内のフロートは `[H]` のみ
- 図表・式・本文で同じ量記号を使う（例: 電位は φ で統一、V は電圧）
- 章をまたぐ記号の不一致は監査対象（例: 単位ベクトルの記法、σ の用途）
- 提出時は log・EPS 原図含む全ファイルを納品（aux 類も消さない）

## 9. 章執筆の標準手順（エージェント向け）

1. このファイルと `章立て案A_節構成_20260612.md` の該当章を読む
2. 出典タグ（〔現n.m〕）の既存草稿 `textbook/chapters/` を読み、移植・改稿
3. スライド画像参照（`\fbox{\includegraphics{slide/...}}`）は**全廃**し、必要な図は scripts で EPS を新規生成
4. 例題・問は講義リポ（electromagnetic-theory）の演習を素材にしてよい
5. `book/build.sh` でエラー0を確認してから完了報告（オーバーフル警告も報告に含める）
6. コミットはしない（メインセッションが監査後に実施）


## 10. パワエレ教科書 固有の規約

- **理念**: (1)素子が「なぜそう動くか」から機能を導く (2)「エネルギーをどう蓄え、どう放出するか」の一貫視点で全変換回路を説明 (3)仮定（理想スイッチ・定常状態・小リプル）を毎回明示 (4)リプルから素子値を決める設計手順まで下ろす (5)LTspiceで確かめられるようにする
- **本章の現在地**: 第II部の各章冒頭に、序章の「電力変換の地図」(fig0.1)上の現在地と「エネルギーをどこに蓄え、いつ放出するか」の一言を screen 囲みで置く（電磁気本の様式を踏襲）
- **波形図**: LTspiceのシミュレーション結果を素材にする場合も、必ず matplotlib スクリプトで再描画してEPS化する（スクリーンショット貼り込み禁止。データはスクリプト内で式から生成するか、csvを figures/scripts/data/ に置く）
- **回路図**: TikZ（プレーン。circuitikzは使わない — platex+dvips経路の互換性のため）。素子記号は本書内で統一
- **LTspice**: 対応するシミュレーションは ~/power_electronics_oit_jinno/LTspice/ を参照し、章末問題に「LTspiceで確かめよ」型の課題を置いてよい
- **単位検算**: 新しい式が出るたびに「両辺の単位を確かめる練習」を問として置く（講義の看板手法）
- **文体**: である調、句読点「，」「。」。素材（lecture_notes）はです・ます調なので必ず変換する
