# chapter09 LTspice検証メモ

第9章（直流-交流変換 インバータ）で本文が【LTspiceで確かめよ】と明示的に要求するのは章末問題
ex9.6のみ（フルブリッジインバータを正弦波PWMで動かすモデル。負荷RL直列R=10Ω/L=10mH、
搬送波fc=5kHz、変調波fm=50Hz、変調率m=0.8）。他章と同じ絞り込み判断で、この1系統から
バイポーラ変調（`hbridge_pwm_bipolar`）とユニポーラ変調（`hbridge_pwm_unipolar`）の2回路を
作成した（ex9.6(c)がバイポーラ/ユニポーラの比較を明示的に要求するため、第7章の
zener/opamp2回路比較と同じ理由での複数回路作成）。

## 回路構成

標準的なフルブリッジ（4スイッチ、対角配置）+ RL直列負荷。E=100Vは本文がex9.6で明示しない値の
ため、同じ章のrei9.3例題・ex9.1(c)と同じ100Vを採用した（教科書内で既に使われている値を流用、
恣意的な新規値の導入を避けた）。

- **バイポーラ**: 対角ペア(S1,S4)と(S2,S3)を単一の比較器出力(`ctrlA`/`ctrlAn`)で連動させる。
  出力`Vab`は+Eと-Eの2値のみ。
- **ユニポーラ**: 左レグ(S1,S2)は+v_ref、右レグ(S3,S4)は-v_refで独立制御(`ctrlA/ctrlAn`と
  `ctrlB/ctrlBn`)。出力`Vab`は+E,0,-Eの3値。

## 新規手法: PWM比較器をvoltageシンボル流用のbehavioral sourceで実装

正弦波PWMの生成には「三角搬送波と正弦波基準波を比較する」という連続時間の論理演算が必要で、
既存の`sw`用PULSE電圧源だけでは表現できない。本章で新たに、`voltage`シンボルを流用して
ngspice behavioral source（B-element, `SYMATTR Value V=<式>`）を実装する手法を追加した：

- `Bcarr`: 三角搬送波 `V=(2/pi*asin(sin(2*pi*fc*time)))`（`asin(sin(x))`で振幅±1の三角波を
  生成、ngspiceのB-source式パーサーがasin/sin/三項演算子`?:`をサポートすることを`/tmp`相当の
  スクラッチ環境で事前確認済み）
- `Bref`: 正弦波基準波 `V=(m*sin(2*pi*fm*time))`
- `BctrlA`/`BctrlAn`: 比較器 `V=((V(ref)>V(carr))?1:0)` / `V=((V(ref)<V(carr))?1:0)`
  （スイッチの`ctrl+`/`ctrl-`ピンへ named FLAG で無線状に接続、chapter05のPULSE制御と同じ
  「同名FLAGで電気的に結線」パターンを流用）

視覚上はvoltageシンボル（電池記号）で描画されるが実体は制御ブロックであり、opamp/zenerと同じ
「既存シンボルを流用した簡略化」方針である旨をVERIFICATION_NOTES・回路図TEXT両方に明記した。
`asc_to_net.py`のコード変更は不要だった（InstNameを`B<name>`にするだけでSPICE B-element行
`B<name> <net+> <net-> V=<式>`がそのまま生成される、既存の汎用SYMATTR処理で対応できた）。

## 検証ファーストで発見した罠

1. **`.meas ... to=<tstop>`ちょうどの境界でアーティファクトが出る**: `meas tran vmax max vab
   from=20m to=40m`のように測定窓の上端をtran終了時刻ちょうどに合わせると、スイッチング回路
   では最終強制サンプル点で桁違いの異常値（実測: 期待値100Vのところ24847V）が出ることを発見。
   `to=39.9m`のように終了時刻より僅かに手前までの窓にすることで解消した。`tools/README.md`に
   今後スイッチング回路でmeasを使う章向けに明記。
2. **B-source式内で`.param`が直接参照できない場面がある**: `.control`内の`let`式（例:
   `let sinref = sin(2*pi*fm*time)`）は`.param`の値を認識せず`vector fm is not available`で
   失敗する。`let`はngspiceの別スクリプト言語であり、SPICEデッキの`.param`置換とは独立している
   ため。`let`式では数値を直接埋め込む（`sin(2*3.14159265358979*50*time)`）ことで回避した。
   一方、SYMATTR Value等のSPICEカード内では`.param`置換は問題なく機能する（`Bref`の`m`/`fm`
   参照は正常動作）。
3. **`.four`組込みFourier解析は高周波PWM波形を粗いグリッドで過小評価する**: ngspiceの`.four`
   directiveはデフォルトgridsize=200点固定で、5kHz搬送波(20ms基本波周期に対し1周期200µs)を
   解像するには不十分（1搬送波周期あたり2サンプル相当）。実際に基本波振幅を大きく誤った値
   （期待80Vに対し127V）で報告した。代わりに相関積分法（`let ib=vab*sin(wt); meas tran ...
   integ ib`、フーリエ係数を数値積分で直接計算）を使用し、既知振幅の校正テストで完全一致
   （誤差<0.01%）を確認した上で採用した。
4. **`fft`コマンドはゼロパディングでビン間隔がbasic周期に対して非整数倍になり得る**:
   40ms窓（50Hzの厳密2周期）でも、内部でFFT長を2のべき乗に拡張するため実際のビン間隔が
   期待の25Hzではなく約15.26Hzになり、50Hz基本波が単一ビンに収まらずリーク（漏れ）することを
   確認。定量的な基本波振幅の抽出には使わず、「高調波がどの周波数帯に集まるか」という定性的な
   確認（ex9.6(b)(c)の主目的）にのみ使用した。

## ngspice実測（バイポーラ、m=0.8基準ケース）

| 項目 | 実測値 |
|---|---|
| Vab最大/最小 | +100.16V / -100.16V |
| 負荷電流 Irms | 5.401A |
| 負荷電流 Ipeak | 7.839A |

RL直列インピーダンス |Z| = sqrt(10² + (2π×50×0.01)²) ≈ 10.48Ω、基本波電圧振幅mE=80Vからの
理論ピーク電流 80/10.48 ≈ 7.63A で、実測Ipeak=7.839A と概ね整合（スイッチング高調波分の上乗せ
を含むため理論値よりわずかに高いのは妥当）。

### ex9.6(a): バイポーラ変調で電流が正弦波に近づくことの確認

Irms=5.401A、Ipeak/√2=5.543A で近い値（両者が一致するのは純粋な正弦波の場合。差は
スイッチングリプル分）。負荷のLがリプル電流を平滑化し、電流波形が電圧波形よりも正弦波に
近いことを定量的に確認（sec9.5本文の主張どおり）。

### ex9.6(b): 出力電圧のFFT、高調波がfc付近に集まることの確認

`linearize`+`fft`でVabの振幅スペクトルを取得。上位ピークはfc=5kHz近傍（4974〜5096Hz帯）と
基本波近傍（50Hz帯）にほぼ同程度の大きさで現れ、2fc=10kHz近傍にも副次的なピーク群が見られた。
50Hz近傍・数百Hz域には目立ったピークが無く、「高調波は搬送波周波数fcの付近に固まる」という
sec9.5の主張を定性的に確認した（絶対値の校正は上記の罠4によりFFTでは行わず、相関積分法で
別途実施）。

### ex9.6(d): 変調率mと基本波振幅の関係(相関積分で実測)

| m | 実測 V̂1 [V] | 理論 mE [V] | 一致率 |
|---|---|---|---|
| 0.5 | 49.93 | 50.0 | 99.9% |
| 0.8 | 79.84 | 80.0 | 99.8% |
| 1.0 | 100.0 | 100.0 | 100.0% |
| 1.2（過変調） | 110.23 | 120.0（線形延長） | **91.9%（下方乖離）** |

m≤1では線形関係V̂1=mEをすべて99.8%以上の精度で確認。m=1.2では線形予測(120V)から明確に下方
乖離（110.23V）し、sec9.5の「注意」が述べる過変調（m>1で線形から外れ、方形波の基本波振幅
4E/π≈127.3Vへ漸近する）を定量的に確認した。110.23Vは100V(m=1)と127.3V(方形波極限)の間に
収まっており、過変調の物理的挙動と整合する。

## ngspice実測（ユニポーラ、m=0.8、バイポーラとの比較）

| 項目 | バイポーラ | ユニポーラ |
|---|---|---|
| Vab最大/最小 | +100.16V/-100.16V | +100.04V/-100.04V |
| 負荷電流 Irms | 5.401A | 5.392A |
| 負荷電流 Ipeak | 7.839A | 7.714A |
| 基本波振幅(相関積分) | 79.84V | 79.81V |
| Vab平均値(DC成分) | - | 0.0049V(≈0、DCオフセット無しを確認) |

### ex9.6(c): バイポーラ/ユニポーラの高調波帯とリプルの比較

基本波振幅はバイポーラ・ユニポーラでほぼ一致（79.84V vs 79.81V、差0.04%）。「振幅は変調率で
決まり変調方式には依存しない」という定義9.3の帰結を定量的に確認した。

FFTスペクトルでは明確な対比が得られた: バイポーラはfc=5kHz近傍に強いピーク群（最大48.7、
5005Hz）を持つのに対し、ユニポーラは**同じ5kHz近傍に目立つピークが無く**、最初の顕著な
ピーク群は2fc=10kHz近傍（最大19.09、9949Hz）に移っていた。これはsec9.5本文の
「ユニポーラでは実効的なスイッチング周波数が2倍になり、高調波は2fc付近に移る」という主張と
完全に整合する、明確で定量的な確認結果である。

負荷電流のリプルはIpeak基準でバイポーラ7.839A→ユニポーラ7.714Aとわずかに減少しており、
高調波が高い周波数へ押しやられたことでLによる平滑化がより効きやすくなった（sec9.5本文の
「後段のLCフィルタで取り除くのも容易」という主張と整合）ことも確認できた。

## 環境上の限界（既存の申し送りを継続）

本機にwine/実LTspiceが無いため、シンボルアートワークは引き続き本パイプライン独自の`.asy`。
`Bcarr`/`Bref`等の制御ブロックはvoltageシンボルを流用しており、視覚上は電圧源記号だが実体は
比較器・搬送波生成という制御ロジックである点は上記の通り明記済み。`.options reltol/gmin`の
緩和はスイッチング回路の収束のための現実的な調整であり、実LTspiceのデフォルト収束挙動とは
厳密には一致しない可能性がある。
