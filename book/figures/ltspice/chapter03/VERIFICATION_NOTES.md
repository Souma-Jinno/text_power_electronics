# chapter03 LTspice検証メモ

第3章（デバイス特性とデバイス選択）は物理量から式を導く章で、耐圧-オン抵抗トレードオフ
（sec3.2, rei3.1/rei3.3）やバリガ性能指数（sec3.5）は電卓で完結する解析計算であり、
「回路そのもの」を要求する対象ではない（依頼書見込み表の「3-5回路」はch02同様に概算で、
本文優先で1回路に絞った。JFETが消えたch02と同じ判断パターン）。
本章で唯一「回路」として意味を持つのは sec3.3・例題3.2 のスイッチング損失の物理
（電圧・電流の重なりが損失を生む）であり、これをクランプ誘導性ダブルパルス試験
（switching_loss_double_pulse）で検証した。

## switching_loss_double_pulse（sec3.3、例題3.2の物理的検証）

回路: V1=100V(vtop) - L1=50µH - M1(スイッチ, ドレイン=sw, `.model SWMOS NMOS(LEVEL=1 VTO=3 KP=2)`、
ch02のmosfet_output_charと同じモデル定数)。D1はL1に並列の還流ダイオード(anode=sw, cathode=vtop,
`.model DFW D(Is=1e-9 N=1.0 Rs=0.001)`)。Coss=470pF(sw-GND間、後述)。ゲートはPWL電圧源で
ダブルパルス駆動: 1発目オン5µsでL1電流を約10Aまで直線的に立ち上げ→1µs休止でターンオフ過渡を
観測→2発目オンでターンオン過渡を同じ電流レベルで再観測。スイッチ電流はKCL
`i_sw = i(L1) - i(D1)` で後処理により導出、`v(sw)*i_sw` の時間積分でE_on/E_offを実測。

**モデル化上の工夫（本文にはない追加要素、正直に記載）**:

1. **`save all @d1[id]` の追加が必須だった**: 当初の実装（コミット前に発見）は`.control`ブロックで
   `wrdata`に`@d1[id]`を渡すだけで、明示的な`save`文がなかった。この状態でシミュレーションすると
   `@d1[id]`はトランジェント解析中一切更新されず、DC動作点解析（t=0時点）の値
   （14.1930A、これ自体もsw=100V・D1逆バイアス時の内部演算誤差起源の非物理値）に**全時刻で固定**
   されたまま出力されるバグがあった。この状態でE_on/E_offを計算すると、i_sw=i(L1)-i(D1)の
   i(D1)が常に14.19Aの定数になるため、スイッチオン期間中も常時i_sw≈-14Aという物理的にありえない
   （導通中のスイッチ電流がマイナス）値が出て、積分結果も無意味になっていた。
   `.control`ブロック冒頭に`save all @d1[id]`を追加してから`tran`を実行することで解決し、
   `i(d1)`がオン期間中≈0A・還流期間中≈i(L1)と一致する物理的に正しい挙動になった
   （rectenna broadband_loopの`memory: zimaboard-oom-sessions`ではなく、同リポの過去記録
   「各ダイオードの平均\|電流\|をngspice実測（`@dX[id]`, run前にsave）」と同じ落とし穴で、
   艦を跨いで再発した既知の罠として記録する）。
2. **Coss=470pF(sw-GND間)を追加した**: 当初の回路にはこの寄生容量がなく、ノード"sw"は
   L1(インダクタ、高インピーダンス)・D1・M1ドレインのみに接続された「容量ゼロのノード」だった。
   このためM1が理想的な有限勾配なしで瞬時にオフする瞬間、ノード電圧に物理的な「慣性」が
   全くなく、ngspiceのニュートン法反復が収束せずD1の電流が最大**約109万A**という明白な
   数値破綻値を出力する現象が発生した（**数値創作ではなく、収束破綻の検出**。監査で発見）。
   実デバイスには必ず出力容量Coss・配線浮遊容量が存在するため、その物理的近似として
   Coss=470pFを追加（100pFでは残存する小さな収束アーチファクトが解消しなかったため470pFへ増量）。
   これにより収束は安定したが、下記のとおりこの値がターンオフ速度そのものを支配してしまう
   副作用がある（次項）。

**ngspice出力（実測、`analyze_dpt.py`で後処理）**:
```
E_off (turn-off, window 5100-5300ns): E ≈ 3.43 uJ
E_on  (turn-on,  window 6200-6400ns): E ≈ 8.83 uJ
Coss充放電の理論値: dt = Coss*V/I = 470p*100/10.1 ≈ 4.65 ns, 0.5*Coss*V^2 ≈ 2.35 uJ
```

**本文との突き合わせ（定性的には確認、定量的には再現できず・正直に記載）**:

- **定性的に確認できた核心**: 「切り替えの途中で電圧と電流が同時に非ゼロになり、その間
  `p=vi>0`の電力が熱になる」というsec3.3の核心的主張どおり、ターンオン・ターンオフとも
  正の実測エネルギー（E_off≈3.4µJ, E_on≈8.8µJ）が得られた。切り替え前後（オン中は
  v(sw)≈0.4V, オフ中はi_sw≈0A）ではp≈0であることも波形から確認でき、「オン・オフでは
  損失なし、切り替えの瞬間だけ損失が出る」という定性的構図は再現できている。
- **定量的には再現できなかった**: 例題3.2はE_on+E_off=50µJ（t_on=t_off=50ns、式3.8/3.9の
  仮定）だが、本シミュレーションはE_on+E_off≈12.3µJで、本文の約1/4しかない。原因は
  **モデルの限界**であり数値創作で埋めていない: 本回路のLEVEL=1 MOSFETモデルにはゲート容量
  （Miller効果を生むCgd）が実装されておらず、ターンオフの実際の速度は
  「ゲート駆動の遅さ」ではなく「上記(2)で追加したCoss=470pFの充放電速度」で決まってしまう
  （dt≈4.65ns、本文が仮定する50nsの約1/10）。したがってE=0.5*V*I*dtで見ても
  0.5*100*10.1*4.65ns≈2.35µJとなり、本文の25µJという値とは前提とする切り替え時間が
  そもそも違う。**本文のt_on=t_off=50nsという数値は与えられた仮定（実測値の代入）であり、
  本回路はそれを独立に再現する回路（正確なゲート駆動遅延回路）を持っていない**ため、
  この数値の一致は最初から意図していない（ch02のigbt_equivalent_vonと同種の「モデルの
  意図的な簡略化により、定性的挙動は確認できるが定量値は本文と別物になる」ケース）。
- 実測値には収束アーチファクト由来の残差も残る（ターンオフ直後に一瞬だけi_sw が
  -2.15Aまで振れる1サンプル分のオーバーシュートが`analyze_dpt.py`で確認できる）。
  Coss=470pFでほぼ解消したが完全なゼロ交差の1サンプルには残っており、E_off≈3.4µJの
  うち一部（Coss理論値2.35µJとの差1.1µJ弱）はこの残差ダイオードRs損失＋数値誤差を
  含む可能性がある。E_onの方はこの種の振動が見られず、Coss理論値との整合は取れていない
  （8.83µJ vs 2.35µJ、ターンオン側はダイオードの逆回復的な追加電流成分が乗るため単純な
  Coss充放電モデルより大きくなるのは物理的に自然だが、定量一致は主張しない）。

## 全体所感・妥協点まとめ

- 本章はch02までと異なり「回路の再現」よりも「損失物理の定性的デモンストレーション」が
  目的の章であり、その核心（切り替え中の電圧電流重なり→正の実測エネルギー）は確認できた。
- 監査で発見した2件の欠陥（`@d1[id]`のsave漏れによる定数固定バグ、Coss=0による収束破綻）は
  いずれも「実装エージェントの成果物をメインセッションが監査してから初めてコミットする」
  規約（CLAUDE.md）どおり、コミット前に検出・修正した。前者は艦を跨いで既知だった罠
  （rectenna broadband_loopの過去記録）が本リポでも再発した例として明記した。
- E_on/E_off の絶対値は本文の例題3.2の数値（50µJ、t_on=t_off=50ns仮定）と一致しない。
  これはモデルの意図的な単純化（LEVEL=1・ゲート容量なし）の帰結であり、数値を合わせるための
  パラメータ調整（Coss値の恣意的なチューニング等）は行っていない。
