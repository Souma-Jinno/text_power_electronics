# 第5章 LTspice図 3点監査（2026-07-18）

対象: `buck_chopper.svg` / `boost_chopper.svg` / `buckboost_chopper.svg`
（`book/figures/ltspice/chapter05/`）。監査ツール: `tools/svg_preview.py`（本セッションで
回転(`rotate()`)未対応のバグを修正済み、下記「発見・修正」参照）。

## 発見・修正（監査ツール自体のバグ）

`svg_preview.py` はシンボルの `<g transform="translate(x,y) rotate(r)">` のうち
`rotate` 部分を解析だけして描画に反映していなかった（`tx,ty` だけ使用、回転を無視）。
このため `buck_chopper` の還流ダイオード D1（`SYMBOL diode ... R180`）が実際の座標から
外れた位置に浮いて見えるプレビューが生成され、一見「配線されていない」ように誤診断
されかけた。実SVG側（`ltspice_to_svg`が出力する`translate(...) rotate(180)`）と
`asc_to_net.py`の接続チェック・ngspice接続結果は最初から正しく、プレビューツール側の
表示バグだった。`rotate_pt()`を追加しline/rect/ellipse/text座標に回転を適用するよう修正
（`tools/svg_preview.py`）。今後R90/R270等を使う章（6章以降、変圧器・ブリッジ整流等）でも
同じ検証精度が効くようになった副産物。

## 3点監査

### (i) フォントサイズ
3回路とも部品ラベル・値・ディレクティブは24px統一（既存章と同一）。縮小・潰れなし。

### (ii) はみ出し・重なり
- 初回生成時、タイトルコメント(y=-120)と`.tran`ディレクティブ(当初x=400,y=-120に配置)が
  横方向に衝突し文字が重なって読めなかった（`buck_chopper`で発見、`D=0.5.tranHzL30mH`のような
  文字化け状の重なりとして視認）。3回路とも`.tran`をディレクティブブロック末尾(y=680)へ移動し解消。
  修正後、viewBox幅が縮小（例: buck 1433.1→944.3、余分な余白も同時に解消）。
- 回路本体（V1〜R1、制御源V2）同士の重なりなし。GNDフラグ・ctrlラベルの位置も部品と衝突なし。

### (iii) 余白バランス
制御源V2サブ回路が左下に独立配置されるため、中央〜右下にまとまった空白が生じるが、
既存章(第1〜4章)と同じレイアウト密度で過大な余白・偏りではないと判断。

## 監査結果: 3回路ともNGなし。
